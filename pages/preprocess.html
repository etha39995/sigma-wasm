About SmolVLM & WASM Preprocessing

This demo showcases WebAssembly (WASM) preprocessing for SmolVLM-500M-Instruct, a compact vision-language model from Hugging Face. SmolVLM processes both images and text to generate multimodal outputs like image captioning and visual question answering.

Architecture: This preprocessing step uses Rust-compiled WASM to efficiently resize images and tokenize text before feeding them to SmolVLM. In a full implementation, the preprocessed data would be passed to SmolVLM running via JavaScript frameworks (like Transformers.js) on WebGPU for inference, enabling fully client-side, on-device AI without server calls.

Image Processing: SmolVLM uses 384×384 image patches encoded as 81 visual tokens. Larger images are divided into patches, each processed separately. This demo resizes images to 384×384 using high-quality Lanczos3 filtering in WASM for optimal preprocessing performance.
